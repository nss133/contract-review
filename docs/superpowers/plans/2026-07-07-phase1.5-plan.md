# Phase 1.5 구현 계획: 증적 중심 원자 체크리스트

> **For agentic workers:** REQUIRED SUB-SKILL: superpowers:subagent-driven-development. 스펙: `docs/superpowers/specs/2026-07-07-phase1.5-evidence-first-redesign.md` (스키마 v2·검증 규칙·UI 정의는 스펙이 기준). 기존 코드베이스가 참조 구현이다 — 각 태스크는 관련 기존 파일을 Read 후 수정한다.

**Goal:** checkpoints(guidance 중심 28개) → checks(원자 quote 중심 ~100개)로 전환하고 UI를 체크리스트 표 중심으로 개편, 풀 빌드까지.

**전제 (검증된 사실):** 테스트 기준선 Python 22 + JS 8. `node --test`는 무인자 실행. DB article_ref는 '제N조'/'제N조(제목)' 혼재(lookup_article 4단계 폴백 있음). 조문 text는 `######` 접두 포함 — quote 대조 시 공백·개행 정규화 필수.

---

### Task A: 스키마 v2 — validate.py 개편 (TDD)

**Files:** Modify `build/validate.py`, `tests/conftest.py`, `tests/test_validate.py`, `knowledge/schema.md`

- `checkpoints` → `checks` 키. check 필수 필드: {id, check, severity, norm_type, basis}
- norm_type ∈ {강행, 임의, 추정, 간주, 실무} / basis ∈ {statute, practice}
- basis=statute → sources 1개 이상 + sources[0].quote 필수 (없으면 ValidationError). 각 source: {law, article, verified} 필수, {clause, quote} 선택
- basis=practice → sources 빈 리스트 허용
- guidance 필드 발견 시 ValidationError("guidance는 폐지됨 — note 사용")
- note는 선택·문자열. 기존 검증(id 유일, module 선언, patterns 정규식 컴파일, 빈 meta/checks 방어) 전부 유지
- conftest 픽스처를 v2 형태로 교체. 테스트: 기존 10개 상당 + 신규(quote 누락 statute 거부, norm_type 오류 거부, guidance 잔존 거부, practice의 빈 sources 허용) → validate 계열 ~14개
- schema.md를 스펙 v2 기준으로 재작성
- 커밋: "feat!: 스키마 v2 — 원자 checks·quote 필수·guidance 폐지"

### Task B: enrich.py v2 — quote 대조 검증 (TDD)

**Files:** Modify `build/enrich.py`, `tests/test_enrich.py`

- `_normalize(s)`: 공백·개행 연속을 단일 공백으로, `#` 제거, strip — quote와 조문 text 양쪽에 적용
- source 처리: lookup_article(기존 4단계 폴백 유지)로 원문 획득 후:
  - 원문 없음 → status=`missing` + 경고
  - quote 있음 & normalize 후 원문에 부분문자열로 존재 → status=`quote_ok`, `verified_flag`(true/false) 별도 유지
  - quote 있음 & 불일치 → status=`quote_mismatch` + 경고 "{id}: quote 문언이 {law} {article} 원문과 불일치"
  - quote 없음(비첫번째 source) → 기존처럼 text 첨부, status=`quote_ok` 취급하지 않고 `no_quote`
- text 첨부 시 `######` 등 마크다운 헤더 접두 제거 (표시 품질 — 이번에 해결)
- news_refs 로직 유지. 테스트: 픽스처 DB로 quote_ok/quote_mismatch/missing/no_quote/정규화(공백 차이 quote 매칭) 검증 → enrich 계열 ~14개
- 커밋: "feat: enrich v2 — quote 원문 대조 검증 + 조문 마크다운 접두 정리"

### Task C: 지식 원자 분해 — common.yaml + outsourcing.yaml 재작성 (콘텐츠, 법무 하네스 적용)

**Files:** Rewrite `knowledge/common.yaml`, `knowledge/types/outsourcing.yaml`

- 기존 파일과 Phase 1 검증 성과(조번호 보정 내역)를 참조하되, **각 항목의 quote는 반드시 DB에서 조문 원문을 SELECT해 읽고 그 문언에서 발췌**한다 (기억·추정 발췌 금지)
- 분해 기준: 조문 항·호 1개 = check 1개. ID는 `{기존ID}-{n}` (예: OUT-09-1). 인벤토리:
  - 개보법 26조①1~3호+②~⑦, 시행령 28조①1~5호+②~④ → M-PRIV ~15개
  - 신정법 17조①③④⑤⑦ → ~5개
  - 업무위탁규정 3조①~⑤·3조의2·4조①②·5조 → M-CORE ~10개
  - 전자금융감독규정 60조 주요 호·14조의2①②④⑦ → M-IT/M-CLOUD ~12개
  - 금소법 44·45조, 보험업법 83조 → M-SOLICIT ~5개
  - 파견법(DB 미수록) → M-DISPATCH 2~3개, quote 없이 basis=statute 예외 불가하므로 **basis=practice + note에 "(파견법 제6조의2·제7조 — 원문 미확인, vault 확인 필요)"**
  - CMN 10개 → 원자 분해 (CMN-05→민법 393조 quote 등 statute 가능한 것만 statute, 나머지 practice) → ~15개
- 목표 총량 80~110개. norm_type은 원문 어미("하여야 한다"/"할 수 있다"/"본다")로 판정
- triggers: 원자 항목별 구체 키워드 (1글자 금지, Phase 1 과매칭 교훈 적용)
- 검증 게이트: load_knowledge OK + enrich 스모크에서 quote_mismatch 0건·missing은 파견법 외 0건 + pytest 회귀 없음
- 커밋: "feat: 지식 원자 분해 — 공통+업무위탁 ~N개 (quote 전건 DB 발췌, verified:false)"

### Task D: extract_candidates.py — 대량 생성 도구

**Files:** Create `build/extract_candidates.py`, `tests/test_extract.py`

- CLI: `python3 build/extract_candidates.py <law_name> [--article 제N조]` → 해당 법령의 열거형 조문("다음 각 호"+"포함"/"하여야 한다" 패턴)에서 호 단위 분해 → 후보 YAML(checks 형태, check=호 문언 그대로+"— 질문형 정제 필요", quote=호 문언, triggers 빈, basis=statute, verified:false)을 stdout 출력
- 호 분해 정규식: `\n?\s*\d+\.\s` 경계 분리 (조문 text 구조 실측 후 조정)
- 테스트 2~3개(픽스처 DB로 호 분해 정확성). Phase 2 확장용 도구이므로 완성도는 "후보 생성기" 수준이면 충분 — 산출물이 그대로 앱에 들어가지 않음
- 커밋: "feat: 체크리스트 후보 대량 추출 도구 (extract_candidates)"

### Task E: UI v2 — 체크리스트 표 중심 개편

**Files:** Modify `src/app.js`, `src/template.html`, `src/style.css` (segmenter/matcher/docx는 무변경)

- matcher.js의 analyze는 그대로 사용 (checks 배열도 동일 구조로 동작 — cp.id/triggers/module/absence_check 필드 유지됨)
- 주 탭 "체크리스트": `<table>` 행 = check. 열: 매칭배지 / ID / check 질문 / 심각도 / 규범 / 근거(law 제N조 항호 + 증적배지). 필터 셀렉트 3개(모듈·심각도·매칭상태) + 검색 input
- 증적 배지: quote_ok+verified→`원문확인`(녹) / quote_ok+!verified→`원문 미대조`(황) / quote_mismatch→`문언 불일치`(적) / missing→`원문 미확인`(적) / practice→`실무`(회)
- 행 클릭 확장: quote 블록(강조 표시) → 전체 조문 원문(마크다운 접두 제거된) → 매칭 시 해당 계약 조항 발췌 + "추정 매칭 — 직접 확인 필요" 문구
- 분석 플로우 유지(입력→유형→스크리닝→표 갱신). 미검출(absence_check & 미매칭) 행을 표 상단 고정 + ✗ 강조. 매칭 배지 "✓ 제5조 (추정)"
- 조항별 뷰: 보조 탭으로 유지(기존 split 재사용, renderCard를 checks 구조에 맞게 수정 — guidance 제거, quote 표시)
- 리포트 탭: 인쇄용 점검표 유지(check 질문 + 체크박스 + 근거 표기), localStorage 유지
- 검증: node --check 전 파일 + DOM id 전수 대조 + 빌드 후 스모크
- 커밋: "feat!: UI v2 — 증적 중심 체크리스트 표"
