## 수렴점 (3안 일치 — 사실상 확정 가능)
1. **횡단 규제모듈 풀**: X-PII(개인·신용정보)·X-EFIN(전자금융)·X-CLOUD·X-SUB(하도급)·X-INSMOD(모집)·X-ASSET(자산운용) — 현행 4~5벌 중복의 단일화. 중복 순삭제 ~40 check(275→~235).
2. 유형 수 7~9. 화해·지분·NDA(정보)는 독립 유지.
3. 변경합의서·연장계약은 유형이 아니라 원거래 재감지 대상.
4. 미분류는 공통 검토 폴백 + 사람 질의(추측 귀속 금지).

## 종합 추천안 (리스크 프레임 기본 + 감지 안정성 보정) — 9유형 + 횡단 ~12모듈

| # | 유형 | 근거 |
|---|---|---|
| 1 | 조달(물품·용역·시스템·임대차) | 안2·안3 통합안. 검수·하자·대금·해지 체크셋 동일. IT 특수성은 X-EFIN·X-CLOUD·X-IP 모듈이 커버 |
| 2 | 업무위탁(지속 기능수행) | 안2 분리 채택 — 금융회사 업무위탁규정 체크셋(위탁보고·재위탁·감독권)은 콜센터엔 맞고 렌탈엔 안 맞음. 통합하면 현행 오탐 재발 |
| 3 | 여신·담보(대주) | 안1 분리 유지 — 대출·질권·PF 어휘는 감지 안정적, 체크셋(담보·기한이익상실)도 투자와 상이 |
| 4 | 투자·운용(펀드·일임·신탁·파생) | 상동 |
| 5 | 판매·모집채널(GA·방카·퇴직연금 위탁판매) | 안2 분리 채택 — 모집규제·수수료·금소법 체크셋은 채널 계약 고유 |
| 6 | 사업제휴(멤버십·공동사업·상표·MOU) | 안2 분리 채택 |
| 7 | 정보·비밀보호(NDA 확장) | 안2 — NDA+단독 개인정보위탁·정보제공 계약 |
| 8 | 지분·조직 | 3안 일치 유지 |
| 9 | 화해·분쟁종결 | 3안 일치 유지 |

**쟁점 해소 방식**:
- 예금질권(안1 담보 vs 안2 채널): **담보 checks(FIN-SEC)를 X-SEC 횡단모듈로 승격** → 채널 유형+X-SEC 병치로 양쪽 리스크 커버. 2축 구조가 유형 논쟁 자체를 해소.
- 재보험: 유형은 제휴/채널 인접(협약 형태), 실체는 X-REINS 횡단모듈이 견인(출재한도·신용위험·상계).
- 신설 횡단 후보: X-RELATED(대주주·계열사 거래제한, 보험업법 §111 — 미래에셋 계열 거래 다수인데 현행 공백), X-CROSS(국외·준거법 — ISDA·해외건 공백).

## Phase C 계획(확정 후)
1. 횡단모듈 풀 구축(knowledge/modules/cross.yaml 신설, 중복 4벌→1벌 통합 — canonical은 최신·최다 벌 기준)
2. 유형 yaml 재편(9유형) + detect_keywords를 실계약명 어휘로 재구축(안1 실측 빈도 활용)
3. **⑤ 리스크 재검수 동시 수행**: 전 check에 회사 리스크 관점 severity 재조정(severity_override), 신규 리스크 체크(배상 상한·일방해지·자동갱신·면책 불균형·계열사 거래), 중립 제한 플래그
4. 하위호환: 구 type_id 매핑 테이블, 골드셋 7건 재라벨 + 유형별 대표 케이스 확장(9유형×2)
5. Phase D: 6,631 계약명 전량 분류 시뮬레이션 — 감지율·오분류율·미분류율 수치 보고
