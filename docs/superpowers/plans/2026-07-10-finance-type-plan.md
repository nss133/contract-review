# finance(금융·여신) 유형 구축 계획

> 실데이터 741건(대출·담보·여신·이행보증) 근거. #8 유형세분화 2단계. 보험사=대주(빌려주는 자) 관점.

## 범위 (실데이터 분해)

- **여신·대출**(PF·부동산개발·일반대출·리파이낸싱): ~330건. 보험사가 대주.
- **담보**(근질권·저당·양도담보·채권양도): 392건 최다. 특히 **GA 예금질권설정**(영업보증 성격) 대량.
- **이행보증·보증보험**: 35건.
- **유동화**(ABL·MBS·주택저당채권양수도): 19건.

## 유형 구조 (모듈 = 규제 레짐)

type_id: finance / type_name: 금융·여신 / detect_keywords: [대출, 대출약정, 여신, 융자, 담보, 근질권, 질권, 저당, 채권양수도, ABL, 리파이낸싱, PF, 이행보증, 보증보험, 대주]

- **M-LOAN**(여신·대출, always_on): 대출금액·이자율·변제기·기한이익상실·지연손해금·상환방법
- **M-SEC**(담보): 질권/저당/양도담보 설정·대항요건·실행·순위
- **M-GUAR**(보증): 보증범위·연대보증·보증인 최고검색항변·구상
- **M-ASSET**(보험업 자산운용 규제, suggest): 보험업법 자산운용 한도·타인대출 제한

## 구현 순서
1. 법령 조문 korean-law MCP 조달 → external_laws.json seed → extract_snapshot으로 스냅샷 편입.
2. knowledge/types/finance.yaml 작성(모듈·checks, quote는 DB원문 발췌 verified:false).
3. validate·enrich 통과(quote_ok), 유형감지 조정(detectType), 스냅샷 재현성.
4. seed 검수(quote↔원문)로 verified 승급.
5. 풀빌드·회귀·브라우저. 커밋→feat/finance-type→main.

## 판단 필요
- GA 예금질권(영업보증)은 담보이나 성격이 다름 — M-SEC 하위 or 별도 태그.
- 보험업 자산운용 규제(§104~)는 계약서 조항 매칭보다 "이 대출이 자산운용 한도 내인지" 정책판단 — check로 넣되 absence_check 신중.
- 이자제한법 최고이자율: 법인 여신엔 적용제외 다수 — note로 한정.
