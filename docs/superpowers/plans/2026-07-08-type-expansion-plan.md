## 데이터 조달 방식 (핵심 신규)

DB에 없는 법령은 **korean-law MCP(get_law_text)로 조문 전문을 가져와 `data/law_snapshot.sqlite`에 삽입**한다. 스냅샷이 확장 가능한 데이터 계층이 됨.
- `build/fetch_law.py`(신규): (law_name, mst, 조문리스트) → 조문 전문 조회 → snapshot의 law_articles에 upsert. article_ref는 "제N조" 형식, text는 정제된 원문.
- enrich.lookup_article이 snapshot을 그대로 조회하므로 코드 변경 없음.
- 조달 대상은 각 유형 핵심 조문만(전체 법령 아님). 큐레이션이 인용하는 조문 위주.

## 유형 큐레이션 규격 (업무위탁과 동일 파이프라인)

각 유형 = `knowledge/types/<type_id>.yaml`, 자기완결(meta + checks). Phase 1.5·2 규격 그대로:
- check: 질문형 1문장 / quote: DB(또는 스냅샷) 원문 발췌 / severity: norm_type 규범효력 도출 + severity_basis / triggers: 구체 키워드(1글자 금지) / basis: statute|practice / source_type: law|self_regulation
- 모듈: 유형별 meta.modules. IT외주는 업무위탁의 M-PRIV·M-IT·M-CLOUD 개념 재사용(정의 복사). 단순 유형(NDA·합의서)은 모듈 없이 평면 체크리스트 가능.
- detect_keywords: 유형 간 감지 충돌 최소화(용역/위탁/비밀유지/주주 등 구분되게). 통합 후 detectType 검증.

## 유형별 인벤토리 초안 (큐레이션 시 조정)

### ① 용역·구매 (service-purchase)
- 대금지급·지연이자(하도급법 제13조류, 민법) / 부당특약 금지(하도급법 제3조의4) / 검사·검수 / 하자담보책임(민법 제580조류) / 납품·이행지체·지체상금 / 불공정약관(약관규제법 제6~14조) / 위험부담 / 계약해제
- 공통 체크리스트와 중복 주의(대금·해지 등은 common에 이미 있음 — 유형 특화만)

### ② IT외주·전산용역 (it-outsourcing)
- M-IT: 전산 외주 보안·외주인력 통제(전자금융감독규정 제60조) / M-CLOUD: 클라우드(제14조의2) / M-PRIV: 개인정보 처리위탁(개보법 제26조)
- IT 특화: 산출물 지식재산권 귀속 / 하자·유지보수 / 소스코드 에스크로 / SLA·장애책임 / 검수·인수

### ③ NDA (nda)
- 비밀정보 정의·범위(영업비밀보호법 제2조) / 비밀유지의무·목적외 사용금지 / 존속기간 / 반환·파기 / 예외(공지정보 등) / 침해 시 구제·손해배상(제11조) / 개인정보 포함 시 개보법 교차

### ④ 주주간계약·합의서 (shareholders)
- 주식양도제한(상법 제335조) / 우선매수권·동반매도권(tag/drag) / 이사 지명권 / 의결권 구속 / 신주인수·희석방지 / 교착상태 해소 / 경업금지 / 합의서 일반(민법)

## 실행 웨이브

- **Wave 1**: 데이터 조달(용역·구매·NDA 법령 fetch) + 착수 가능 유형 큐레이션(IT외주·주주간 — DB 완비). fetch_law.py 구축.
- **Wave 2**: 용역·구매·NDA 큐레이션(조달 완료된 스냅샷 기반).
- **Wave 3**: 통합 — detect_keywords 조정, 스냅샷 최종 갱신, 풀빌드, 브라우저 E2E(유형 전환·감지·매칭), 병합.

각 유형 큐레이션 후 콘텐츠 리뷰(quote 진실성·norm_type·원자성). 웨이브 사이 사용자 보고.
